# V7 Phase 129 — Route Inspection Flow

Дата: 2026-05-11

## Что сделано

- В drawer маршрута добавлен верхний блок `Состояние маршрута`.
- Добавлен чеклист `Чеклист маршрута` для оператора.
- Для каждого route class теперь сразу видно:
  - общий вердикт;
  - активный путь;
  - количество доменов в группе;
  - факт route reality;
  - риск утечки;
  - готовность RU_LOCAL для `DIRECT_RU` и `TRUSTED_RU_SENSITIVE`.

## Почему это нужно

До этой фазы маршрут можно было проверить, но оператору приходилось читать несколько таблиц и JSON. Теперь сценарий выглядит как рабочая карточка:

1. Открыть `Маршруты`.
2. Нажать на нужный route class.
3. Увидеть короткий вердикт.
4. Пройти чеклист.
5. Открыть домены, preview, client modes, route reality или логи одной кнопкой.

## Что не изменялось

- Реальная маршрутизация не менялась.
- Пользователи не переключались.
- Kill switch не трогался.
- Policy apply не запускался.
- Серверные скрипты V7 core не переписывались.

## Проверка руками

Админка:

- `http://195.2.79.116:7080/admin-v2`

Сценарий:

1. Открыть `Маршруты`.
2. Нажать `DIRECT_RU`.
3. Проверить блок `Состояние маршрута`.
4. Проверить `Чеклист маршрута`.
5. Нажать `Домены`.
6. Вернуться и нажать `Факт маршрута`.
7. Нажать `TRUSTED_RU_SENSITIVE`.
8. Проверить, что видно объяснение про клиентский RU_LOCAL bypass для пользователей в РФ.
