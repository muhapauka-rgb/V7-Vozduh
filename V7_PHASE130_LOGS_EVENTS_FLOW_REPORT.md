# V7 Phase 130 — Logs And Events Flow

Дата: 2026-05-11

## Что сделано

- В разделе `Логи` добавлен фильтр `Объект / контекст`.
- Лента событий теперь показывает колонку `Что значит`, а не только техническое сообщение.
- Клик по событию открывает drawer с:
  - понятным объяснением;
  - объектом события;
  - подсказкой, что делать дальше;
  - технической записью с уже маскируемыми секретами.
- Логи канала переведены на тот же формат: сводка, ошибки, предупреждения, список событий, клик в деталь.
- Контекстные переходы теперь работают лучше:
  - пользователь → логи по IP;
  - канал → логи по id канала;
  - маршрут → логи по route class;
  - событие → связанные события.

## Что не изменялось

- Сбор логов не менялся.
- Retention/лимиты места не менялись.
- Сырые секреты не выводятся.
- Новые фоновые проверки не добавлялись.
- Нагрузка на сервер не увеличивалась.

## Как проверить руками

Админка:

- `http://195.2.79.116:7080/admin-v2`

Сценарии:

1. Открыть `Логи`.
2. Нажать быстрый фильтр `Каналы`.
3. В поле `Объект / контекст` ввести `awg2` или `vless`.
4. Нажать событие в ленте.
5. Проверить блоки `Что произошло`, `Сводка`, `Сообщение`, `Действия`.
6. Открыть `Каналы`, провалиться в канал, нажать `Logs`.
7. Открыть `Маршруты`, провалиться в `DIRECT_RU` или `TRUSTED_RU_SENSITIVE`, нажать `Route logs`.
