# V7 Phase 70: Profile Delivery Status and History

Дата: 2026-05-08

## Что добавлено

- В админке у пользователя появился блок `Profile Delivery`.
- Одноразовые ссылки теперь имеют безопасный `delivery_id` для управления без показа токена.
- История выдачи показывает статусы:
  - `active`
  - `downloaded`
  - `expired`
  - `revoked`
- Активную ссылку можно отозвать из админки.
- Добавлен admin API:
  - `GET /api/profile-delivery-status`
  - `POST /api/actions/profile-delivery-revoke`

## Безопасность

- Полный одноразовый токен не показывается в статусах и истории.
- В истории виден только `token_hint` и `delivery_id`.
- После скачивания или отзыва путь к профилю удаляется из активной записи.
- Ссылка по-прежнему одноразовая и ограничена TTL.

## Важно по внешнему доступу

Сейчас public-safe endpoint уже отделен логически:

- `/profile-delivery/<token>` не требует входа в админку;
- все `/api/*` остаются за авторизацией;
- полный внешний доступ к админке включать нельзя без отдельного reverse proxy allowlist/auth решения.

Следующий безопасный шаг: выделить наружу только `/profile-delivery/<token>`, а саму админку оставить доступной через SSH tunnel или отдельную защищенную схему.

## Проверки

- Python compile должен проходить.
- `GET /api/profile-delivery-status` не должен раскрывать токены.
- `POST /api/actions/profile-delivery-revoke` должен закрывать только активную ссылку.
- После revoke повторная загрузка по ссылке должна давать 404.
