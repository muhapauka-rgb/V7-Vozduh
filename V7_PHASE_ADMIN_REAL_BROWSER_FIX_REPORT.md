# V7 Admin Real Browser Fix Report

Date: 2026-05-12

## Problem

The public Admin V2 page opened, but stayed in the loading state. API endpoints were healthy, so the failure was inside browser JavaScript rendering.

## Root Cause

Two JavaScript syntax issues existed in the HTML generated by `admin/v7-admin-api`:

- a newline splitter was emitted as a real line break inside a string literal;
- a nested `showTab('logs')` handler lost escaping inside a generated HTML string.

Both caused Chrome to stop executing the admin script before data rendering.

## Fix

- Escaped the newline splitter as `\\n` in generated JavaScript.
- Escaped nested quotes for the logs action button in generated JavaScript.

If an old browser tab still shows the loading state, hard refresh the page or open the admin URL in a new tab so the fixed HTML/JS is loaded.
